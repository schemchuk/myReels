## Що зробити

Виконай завдання (Task 1 → Task 8) із плану **по черзі, у порядку**, не
пропускаючи кроків. Кожне завдання вже містить:
- точні шляхи файлів, які створити/змінити;
- повний код для кожного кроку (нічого вигадувати не потрібно — копіюй
  код із плану як є);
- TDD-послідовність: спочатку тест, переконатись що він падає, потім
  мінімальна реалізація, переконатись що тест проходить;
- точну команду коміту в кінці завдання.

## Правила

1. **Дотримуйся TDD-кроків так, як написано в плані** — не пиши
   реалізацію раніше тесту.
2. **Не змінюй сигнатури/назви**, які вже зафіксовані в плані
   (наприклад, `HeyGenClient.pollUntilComplete`, `runPipeline`,
   `Publisher.publish`) — наступні завдання залежать від цих точних імен.
3. Після кожного завдання виконай **тільки `git commit`** з повідомленням,
   яке вказане в кроці "Commit" цього завдання. Команду коміту бери
   буквально з плану.
4. **НЕ виконуй `git push`** і не змінюй remote/branch. Пуш робить
   користувач власноруч після перевірки.
5. Якщо тест не проходить і причина не очевидна з повідомлення про
   помилку — зупинись і опиши проблему, не вигадуй обхідних рішень, які
   не описані в плані.
6. Завдання 7 (Electron main + renderer) не має автотестів — просто
   виконай кроки 1–6 (написати файли, `npm run build`, commit). Завдання 8
   міститиме ручну E2E-перевірку — її виконає користувач.
