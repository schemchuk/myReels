## Що зробити

Виконай завдання (Task 1 → Task 5) із плану **по черзі, у порядку**, не
пропускаючи кроків. Кожне завдання містить:
- точні шляхи файлів, які створити/змінити;
- повний код для кожного кроку;
- TDD-послідовність: спочатку тест, переконатись що він падає, потім
  мінімальна реалізація, переконатись що тест проходить;
- точну команду коміту в кінці завдання.

## Правила

1. **Дотримуйся TDD-кроків так, як написано в плані** — не пиши
   реалізацію раніше тесту.
2. **Не змінюй сигнатури/назви**, які вже зафіксовані в плані — у Task 5
   вони мають збігатись із тим, що визначено в Task 1–4 (наприклад,
   `prepareSubtitleAssets(text, durationSec, ctaUrl, tempDir)` має бути
   викликана з тими ж аргументами в `pipeline.ts`, що й у тесті).
3. Після кожного завдання виконай `git commit` з повідомленням, яке
   вказане в кроці "Commit" цього завдання.
4. **Виключення з попереднього разу: цього разу, після Task 5 (останнє
   завдання), коли всі тести зелені (`npm test`), виконай `git push`.**
   У всіх попередніх завданнях (1–4) — лише `git commit`, без push.
   Пуш робиться рівно один раз, у самому кінці, після Task 5 Step 8.
5. Якщо тест не проходить і причина не очевидна з повідомлення про
   помилку — зупинись і опиши проблему, не вигадуй обхідних рішень, які
   не описані в плані.
6. Перед Task 5 Step 5 (правки в `index.ts`) подивись на поточний вміст
   файлу — він уже містить `win.webContents.openDevTools(...)` рядок,
   додай новий код навколо нього, не видаляй цей рядок.

## Коли закінчиш

Напиши короткий підсумок: які завдання виконано, чи всі тести зелені
(`npm test`), результат `npm run build`, чи виконався push, і чи є щось,
що відхилилось від плану (і чому).
